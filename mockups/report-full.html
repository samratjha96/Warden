Remediation Suggestions

1
Restrict usage to non-sensitive documents unless legal/compliance approves OpenAI data transfer for this workflow.
2
Enforce egress controls so runtime can only reach approved model endpoints.
3
Use a dedicated, scoped API key and secret manager; avoid .env secrets in shared environments.
4
Disable or sanitize logs for sensitive workloads; securely rotate/delete ./logs artifacts after processing.
5
Add dependency integrity controls: --require-hashes in pip, lockfile workflow, or vetted internal mirror.
6
Prefer running in an isolated container or VM with least privilege and outbound network policy.
7
For maintainers: add SECURITY.md, enable branch protection on main, and formalize vulnerability disclosure handling. Security issues #79 and #80 remain open.
